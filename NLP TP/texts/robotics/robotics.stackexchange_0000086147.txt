What is the "full ROS API"? C++ packages come with auto-generated Doxygen API documentation, but the usefullness of that depends on whether the author has spent some time on it. Python packages can also be accompanied by auto-generated docs, but again, will depend on the author spending time.
Comment by gvdhoorn on 2018-03-25:\

Also why is rtt_ros_integration package related api is not present?

rtt_ros_integration is a metapackage. They don't contain any code, so there is no documentation generated ..

ROS is not one library, it is many packages, each with a separate repository and maintainer, and each of which can be installed separately.
