Comment by tryan on 2021-03-15:
I'm not sure you can merge the scans into a different plane.  Your two side lidars are at z = -0.2, but hokuyo_link_top is at z = 0.29.  For the sake of experimenting, what happens if you change the destination frame to one of the side lidars?

A:

The issue was that the scan_merger node needed to be delayed long ennough for gazebo to initiate.
I got the hint from here.
I don't know if a deplay factor can be added to a node in a launch file. So instead of using a single launch file, I'm using two.
I moved the <node ..... > code above into a separate launch file and launch that after the first one which contains the gazebo node i:e <include file='$(find gazebo_ros)/launch/empty_world.launch'/>.
That way, gazebo is up and running, I launch my launch file containing the scan_merger node, the /scan and /test topics now echo merged data from my hokuyo scan topics.
